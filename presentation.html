What is Tussi?

Tussi is a distributed e-commerce platform built with modern microservices architecture

  • 🔐 User Authentication
  • 📦 Product Catalog Management
  • 🛒 Shopping Cart Functionality
  • 📱 Multi-platform Support (Web + Mobile)
  • ⚡ High Performance & Scalability

Second Prototype Features

  • 🌐 API Gateway Integration
  • 🔒 Enhanced Security (JWT)
  • 🔄 Service Orchestration
  • ⚖️ Load Balancing
  • 📊 Monitoring & Health Checks
  • 📱 Multi-Platform Support

Why "Tussi"?

"Intentionally provocative and disruptive—a metaphor to positively alter shopping experiences, creating emotional, sensory, and memorable interactions."

Target Audience: Young adults interested in unconventional wellness, sustainable fashion, and digital art

Architecture Overview

Component & Connector View

Components (10)

  • 🌐 Web Client
  • 📱 Mobile Client
  • ⚖️ Load Balancer (Nginx)
  • 🚪 API Gateway Service
  • 🔐 Auth Service
  • 📦 Products API
  • 🛒 Cart API
  • 🗄️ Auth Database (PostgreSQL)
  • 🗄️ Products Database (PostgreSQL)
  • 🗄️ Cart Database (MongoDB)

Connectors (10)

HTTPS/HTTP, REST APIs, TCP Database Connections

Layered (Tier & Layer) View

Tier 1: Presentation Web Client (Next.js), Mobile Client (React Native)
Tier 2: Communication API Gateway (Node.js/Express)
Tier 3: Logic Controllers, Services, Models
Tier 4: Data PostgreSQL, MongoDB, Mobile Local Storage

Deployment Structure

Security Scenarios

🛡️ Man-in-the-Middle Attack Protection

TLS/SSL encryption prevents data interception

🚫 API Gateway Bypass Prevention

Private networks isolate backend services

🔒 Frontend Compromise Containment

Network segmentation prevents lateral movement

Performance Scenarios

⚖️ Load Balancing

4 API Gateway replicas with round-robin distribution

📈 Scalability

Horizontal scaling for traffic surges

🛡️ DDoS Protection

Rate limiting and request throttling

Architectural Styles

  • Layered (N-Tier): 4-tier separation of concerns
  • Client-Server: Decoupled client and server components
  • Microservices: Independently deployable services
  • Polyglot: Multiple languages and databases

Key Architectural Patterns

  • 🚪 API Gateway Pattern
  • ⚖️ Load Balancer Pattern
  • 🗄️ Database per Service
  • 🖥️ Server-Side Rendering
  • 🔥 Active Redundancy (Hot Spare)
  • 🔄 Passive Redundancy (Warm Spare)
  • ❄️ Cold Spare (On-Demand)
  • 🔒 Secure Channel Pattern

Deployment

🐳 Docker Containerization


# Build and deploy
docker-compose build
docker-compose up -d

# Scale API Gateway
docker-compose up -d --scale api-gateway=4

📊 Service Ports

Load Balancer: 80/443 | Frontend: 3000 | K6 Testing: 6565

Testing Strategy

📊 Load Testing with K6

7-stage performance testing: Warm-up → Ramp-up → Sustained → Stress → Spike → Recovery → Cool-down

Monitoring & Health Checks


# Health endpoints
curl https://localhost:443/health          # Load Balancer
curl http://localhost:9000/health         # API Gateway
curl http://localhost:8000/health         # Auth Service

# View logs
docker-compose logs -f api-gateway
docker-compose logs -f auth-service
